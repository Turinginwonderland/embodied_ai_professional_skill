## 关键流程

```
docx/xlsx → 结构化提取 → 条目化 → 交叉比对 → 裁决标记 → kb_*.md + kb_delta_*.md
                                                      ↓
                                         synthesis-output-generation 取数 → 报告
```

### 出处核实链路

1. Firecrawl 联网搜索（IDC/央视/新华社/36氪/Fortune Business Insights）
2. 用户补充原始出处（FBI 报告原文、丁文超发言逻辑）
3. 区分硬数据（Fact）与洞察/假说（Insight/Hypothesis）分别归档

## 经验记录

- 市场数据多源口径差异大（FBI 宽口径 vs GGII 窄口径），宜记录差异而非掩盖，作为"产业定义未收敛"的证据
- 专家前瞻性发言（如丁文超 40% GDP）属逻辑推演型假说，不应作为硬数据归档，应归入 `洞察/假说` 类型
- Firecrawl 可用于联网搜索验证，但对会议发言逐字稿等非公开内容需人工补充
