# Skill: Input & Knowledge Evolution (输入与知识演进)

## 1. 技能定位
本技能负责接收新的具身智能相关素材（PPT、研报、新闻、政策），提取事实，并与现有知识库进行比对，完成观点的迭代与"破旧立新"。

本技能是研究助手的"摄取 + 沉淀 + 纠偏"能力层。它吃进素材、把可溯源的事实条目沉淀到持久化记忆层，并维持条目间的冲突裁决。报告撰写不在本技能职责内——那由 `synthesis-output-generation` 按需触发。

## 2. 知识库条目 Schema（持久化记忆层接口）
每条知识库条目必须包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 全局唯一条目 ID，格式 `kb-<来源缩写>-<序号>` | `kb-7.9-001` |
| `内容` | 一句可独立成立的事实或观点 | "VLA 把视觉、语言和当前状态转成机器人动作" |
| `类型` | `事实` / `观点` / `数据` | `事实` |
| `来源` | 可回溯的文件名+章节/DOI/URL | "7.9第一次任务汇报最终版.docx §一.1" |
| `时效` | 来源的发布时间或采集时间 | "2025-07-09" |
| `状态` | `有效` / `已废弃` / `冲突-待裁决` / `待补出处` | `有效` |
| `裁决标签` | `证实` / `互补` / `冲突` / `新增` | `新增` |

条目文件命名：`kb_<来源标识>.md`，统一存放于 `docs/outputs/knowledge_base/`。
冲突裁决变更日志：`docs/outputs/knowledge_base/kb_delta_<日期>.md`。

## 3. 核心工作流
1. **结构化提取**：
   - 提取核心论点、技术路线（如：端到端→分层控制）、关键数据（如：量产成本、传感器方案）。
   - 每条提取必须挂来源、标类型、给条目 ID。
2. **新旧对比分析（比对与纠偏）**：
   - 将新提取观点与已有结论交叉验证。
   - 三类信息：
     - **证实（Confirm）**：增强已有证据 → 在被证实条目的 `关联条目` 中追加新条目 id。
     - **互补（Complement）**：填补空白 → 新增条目，标 `互补`。
     - **冲突/纠偏（Conflict）**：新数据推翻旧观点 → **不二选一删除**，双轨并存：旧条目改 `已废弃` 并注明原因，新条目标 `冲突-待裁决` 或 `有效`。
3. **知识库更新机制（破旧立新）**：
   - 遵循"最新、最权威优先"原则裁决。
   - 依据优先级：官方政策/标准 > 厂商发布/论文 > 券商研报 > 二手新闻。
   - 每次更新在 `kb_delta_<日期>.md` 追加：`[旧id] → [新id] | 变更理由 | 裁决依据来源`。

## 4. 约束条件
- 必须注明出处（某券商研报、官方发布会等）。
- 无充足客观证据前不把猜测当定论，标 `待补出处` 或 `冲突-待裁决`。
- 市场数据若无统计机构/口径/年份，一律标 `待补出处`，**严禁凭空补全数字**。

## 5. 与下游 skill 的接口
- 产出物：`docs/outputs/knowledge_base/kb_*.md` + `kb_delta_*.md`。
- 下游 `synthesis-output-generation` 通过读取上述文件取数，不得绕过本技能直接造数据。

## 6. 演进机制（最新官方信息更新）
- 联网检索最新信息（官方站点、厂商博客、arXiv、政策原文）。
- 依"最新最权威优先"裁决，更新条目并记 `kb_delta`。
- **默认不自动触发**，仅在用户要求更新时执行。

## 7. 原文回溯（预留接口，尚未实现）
本技能维护两层存储，检索策略分层，互不混淆：

| 层 | 存什么 | 路径 | 检索方式 |
|----|--------|------|----------|
| 条目层 | 提炼后的事实/观点，带 8 字段 + 裁决关系 | `docs/outputs/knowledge_base/kb_*.md` | 全量读入 + schema 过滤（按状态/类型/关联条目筛） |
| 原文层 | PPT/PDF/网页快照等原始素材 | `docs/raw_materials/` | 向量检索（语义相似度召回段落） |

**关键原则（为何条目层不进向量库）**：
- 条目层最值钱的不是文本相似度，而是 8 字段 schema 维护的裁决结构。
- 纯向量检索会破坏该结构：`冲突-待裁决`条目需双轨呈现，相似度高的那条会被单独召回；`已废弃`条目语义上仍最相似、却不可引用；演进链（如 `kb-7.9-014`→`kb-7.9-014b`）靠`关联条目`字段维护，语义相似度捕捉不到版本关系。
- 故条目层永远全量读入，向量检索只服务原文层。

**预留接口契约**（条件未满足前保持未实现）：
- 脚本路径：`.claude/skills/input-knowledge-evolution/scripts/search_raw.py`
- 调用形式：`python scripts/search_raw.py "<条目内容或关键词>"` → 返回 `docs/raw_materials/` 中最相关的 N 段原文，每段附来源文件名与位置偏移
- 向量库路径：`docs/outputs/knowledge_base/vector_store/`（chromadb 持久化，gitignore）
- 触发条件：满足其一才启动——`docs/raw_materials/` 原文素材累计超过 ~10 份；或单次研究任务需同时参考 3 份以上原文素材。
- 详细落地步骤见根目录 `newphase2.md`，触发前不预先建库、不预装依赖。
