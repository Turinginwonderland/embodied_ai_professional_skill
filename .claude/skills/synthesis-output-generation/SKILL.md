# Skill: Synthesis & Output Generation (综合与研究输出)

## 1. 技能定位
本技能负责调用已被更新的知识库（`docs/outputs/knowledge_base/`），梳理研究思路，构建研究大纲，并按需输出符合行业标准的深度研究报告。**撰写报告为按需触发的辅助功能，默认不主动生成报告。**

## 2. 与 Input 的数据流接口
- 唯一取数来源：`docs/outputs/knowledge_base/` 下由 `input-knowledge-evolution` 产出的结构化条目。
- 取数时遵循条目标记：`已废弃` 条目不引用；`冲突-待裁决` 条目需双轨呈现。
- 未在知识库登记的事实，不得出现在大纲或报告中。

## 3. 核心工作流
1. **思路梳理（思维链构建）**：
   - 根据研究主题（如：人形机器人手部关节的技术路线演进），梳理从"技术背景→核心痛点→解决方案→市场空间→代表企业"的逻辑链。
2. **大纲构建（MECE原则）**：
   - 产出结构化大纲，确保各章节"相互独立，完全穷尽"（MECE）。
   - 每个大纲节点下批注应调用的核心事实与数据支持（关联条目 ID）。
3. **报告撰写（按需触发）**：
   - 仅在用户明确要求时触发。采用行业分析师或战略专家的专业笔调，语言平实严谨，用数据与事实说话。
   - 技术名词使用准确（如：谐波减速器、灵巧手、空心杯电机等）。

## 4. 约束条件
- 严禁使用未在知识库中登记或证实的虚假数据。
- 复杂报告必须分段生成，每完成一个章节需提示用户确认，避免篇幅过长导致质量下降。
